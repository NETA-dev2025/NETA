NETA User Guide

Complete guide to using the Pan-Neuroendocrine Tumor Atlas

Introduction

NETA is a comprehensive transcriptomic resource for neuroendocrine tumor research. This guide will help you navigate the database, download data, and run analyses.

Quick Overview

  • 75+ datasets from GEO, TCGA, and published studies
  • 2,500+ samples across diverse NET subtypes
  • 3 platforms: Bulk RNA-seq, scRNA-seq, Microarray
  • Ready-to-use count matrices and analysis scripts

Browsing Datasets

Using the Web Interface

  1. Visit the Datasets section
  2. Browse datasets by:
    • Data platform (RNA-seq, scRNA-seq, Microarray)
    • Tissue type (Pancreas, Lung, GI, etc.)
    • Sample size
    • Publication year
  3. Click on a dataset to view details

Dataset Information

Each dataset page includes:

  • Sample characteristics and clinical annotations
  • Platform and processing details
  • Quality control metrics
  • Download links for raw and processed data
  • Links to original publications

Downloading Data

Available Data Files

For each dataset, you can download:

Bulk RNA-seq

  • raw_counts.csv - Raw count matrix (genes × samples)
  • normalized_counts.csv - TPM or FPKM normalized expression
  • sample_metadata.csv - Sample annotations
  • gene_annotations.csv - Gene information

Single-Cell RNA-seq

  • expression_matrix.h5 - HDF5 format expression data
  • cell_metadata.csv - Cell annotations and UMAP coordinates
  • marker_genes.csv - Top markers for each cell type
  • seurat_object.rds - Pre-processed Seurat object (R)

Microarray

  • normalized_expression.csv - Normalized probe intensities
  • sample_metadata.csv - Sample annotations
  • platform_annotation.csv - Probe annotations

Running Analyses

Differential Expression (Bulk RNA-seq)

Using DESeq2

library(DESeq2)

# Load data
counts <- read.csv("GSE98894_raw_counts.csv", row.names=1)
metadata <- read.csv("sample_metadata.csv", row.names=1)

# Create DESeq2 object
dds <- DESeqDataSetFromMatrix(
  countData = counts,
  colData = metadata,
  design = ~ condition
)

# Run analysis
dds <- DESeq(dds)
results <- results(dds)

# Extract significant genes
sig_genes <- subset(results, padj < 0.05)

Using edgeR

library(edgeR)

# Create DGEList object
y <- DGEList(counts = counts)

# Filter low expression genes
keep <- filterByExpr(y)
y <- y[keep, ]

# Normalize
y <- calcNormFactors(y)

# Design matrix
design <- model.matrix(~ condition, data = metadata)

# Estimate dispersion and fit model
y <- estimateDisp(y, design)
fit <- glmQLFit(y, design)
qlf <- glmQLFTest(fit, coef = 2)

# Get results
results <- topTags(qlf, n = Inf)

Single-Cell Analysis

Using Seurat

library(Seurat)

# Load preprocessed object
seurat_obj <- readRDS("seurat_object.rds")

# Or create from scratch
counts <- Read10X_h5("expression_matrix.h5")
seurat_obj <- CreateSeuratObject(counts = counts)

# Standard workflow
seurat_obj <- NormalizeData(seurat_obj)
seurat_obj <- FindVariableFeatures(seurat_obj)
seurat_obj <- ScaleData(seurat_obj)
seurat_obj <- RunPCA(seurat_obj)
seurat_obj <- FindNeighbors(seurat_obj, dims = 1:30)
seurat_obj <- FindClusters(seurat_obj, resolution = 0.5)
seurat_obj <- RunUMAP(seurat_obj, dims = 1:30)

# Visualize
DimPlot(seurat_obj, reduction = "umap")

Pathway Enrichment

library(clusterProfiler)
library(org.Hs.eg.db)

# GO enrichment
ego <- enrichGO(
  gene = sig_genes,
  OrgDb = org.Hs.eg.db,
  ont = "BP",
  pAdjustMethod = "BH",
  pvalueCutoff = 0.05,
  qvalueCutoff = 0.05
)

# KEGG enrichment
ekegg <- enrichKEGG(
  gene = sig_genes,
  organism = 'hsa',
  pvalueCutoff = 0.05
)

# Visualize
dotplot(ego, showCategory = 20)
barplot(ekegg, showCategory = 15)

Data Formats

Count Matrix Format

Count matrices are provided as CSV files:

            Sample1  Sample2  Sample3  Sample4
Gene1          120      145       98      156
Gene2         2340     2156     2401     2289
Gene3            0        2        1        0
Gene4          456      523      478      501

Metadata Format

Sample metadata includes:

  • sample_id: Unique sample identifier
  • condition: Experimental condition or tumor type
  • tissue: Tissue of origin
  • stage: Disease stage (if available)
  • age, sex: Patient demographics
  • batch: Sequencing batch

Frequently Asked Questions

General Questions

Q: Is NETA data free to use?

A: Yes! All data is released under MIT License for both academic and commercial use. Please cite the original data sources.

Q: How often is the database updated?

A: We add new datasets monthly and update existing datasets as new versions become available.

Q: Can I submit my own dataset?

A: Yes! Please email contact@neta-atlas.org with your GEO accession number or contact us for private data sharing.

Technical Questions

Q: Which R version is recommended?

A: R ≥ 4.0.0 is recommended for all analysis scripts.

Q: Are the count matrices normalized?

A: Files named raw_counts.csv contain unnormalized integer counts suitable for DESeq2/edgeR. Files named normalized_counts.csv contain TPM or FPKM values.

Q: Can I use this data with Python?

A: Absolutely! All CSV files can be read with pandas, and single-cell data works with scanpy.
