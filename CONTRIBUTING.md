# Contributing to NETA

Thank you for your interest in contributing to the Pan-Neuroendocrine Tumor Atlas (NETA)! We welcome contributions from the research community.

## Ways to Contribute

### 1. Submit New Datasets

We are always looking to expand our database with high-quality datasets.

**Requirements:**
- Publicly available data (GEO, SRA, or published studies)
- Minimum 3 biological replicates per group
- Raw or minimally processed data available
- Clear sample annotations

**How to submit:**
1. Email us at contact@neta-atlas.org with:
   - Dataset accession number (e.g., GSE12345)
   - Brief description of the study
   - Link to publication (if available)
2. We will review and integrate the dataset within 2-4 weeks

### 2. Report Issues

Found a bug or have a suggestion?

- **Open an issue** on GitHub
- Provide clear description and steps to reproduce
- Include screenshots if applicable
- Tag appropriately (bug, enhancement, documentation)

### 3. Improve Documentation

Help us make NETA more accessible:

- Fix typos or clarify instructions
- Add missing information
- Create new tutorials
- Translate documentation

**Process:**
1. Fork the repository
2. Make your changes
3. Submit a pull request

### 4. Contribute Analysis Scripts

Share your analysis workflows with the community:

- Differential expression pipelines
- Pathway enrichment methods
- Visualization scripts
- Integration workflows

**Requirements:**
- Well-documented code
- Example data or test cases
- Compatible with NETA data format

## Development Guidelines

### Code Style

**R Code:**
- Follow tidyverse style guide
- Use meaningful variable names
- Comment complex operations
- Include error handling

**JavaScript:**
- Use ES6+ syntax
- Follow Airbnb style guide
- Add JSDoc comments

### Commit Messages

Use clear, descriptive commit messages:

```
Add: Feature description
Fix: Bug description
Update: What was updated
Docs: Documentation changes
```

### Pull Request Process

1. **Fork** the repository
2. **Create** a feature branch (`git checkout -b feature/amazing-feature`)
3. **Commit** your changes (`git commit -m 'Add amazing feature'`)
4. **Push** to the branch (`git push origin feature/amazing-feature`)
5. **Open** a Pull Request

**PR Guidelines:**
- Provide clear description of changes
- Reference related issues
- Include before/after comparisons if applicable
- Ensure all tests pass
- Update documentation as needed

## Testing

Before submitting:

- Test your code with multiple datasets
- Verify documentation renders correctly
- Check for broken links
- Ensure browser compatibility (Chrome, Firefox, Safari)

## Questions?

- **Email**: contact@neta-atlas.org
- **GitHub Issues**: For technical questions
- **Discussions**: For general questions and ideas

## Code of Conduct

Be respectful and constructive:

- Use welcoming and inclusive language
- Respect differing viewpoints
- Accept constructive criticism gracefully
- Focus on what is best for the community

## Recognition

All contributors will be acknowledged in:

- CONTRIBUTORS.md file
- Release notes
- Future publications (for significant contributions)

Thank you for helping make NETA better!
